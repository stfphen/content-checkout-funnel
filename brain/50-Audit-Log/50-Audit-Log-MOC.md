# 📓 Audit Log MOC

The project's memory: what happened, why, what's broken, and how to log future work.

## Notes
- [[51-Timeline]] — chronological history of the project (append new entries here).
- [[52-Decision-Log]] — key decisions + rationale (the "why").
- [[53-Known-Issues]] — open risks, security findings, bugs, tech debt.
- [[54-Session-Log-Template]] — copy this to record a future work session.

## How to use this section (for Claude)
After meaningful work:
1. Append a dated bullet to [[51-Timeline]].
2. If you made/changed a decision, add it to [[52-Decision-Log]].
3. If you found or fixed a problem, update [[53-Known-Issues]].
4. For a substantial session, create a session log from [[54-Session-Log-Template]] in this folder (e.g. `55-Session-2026-07-01.md`).
